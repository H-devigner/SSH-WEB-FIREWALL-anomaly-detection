# ELK Live Monitoring With Docker On Windows

This folder runs Elasticsearch, Logstash, and Kibana with Docker Compose on **Windows PowerShell**. It connects Kibana to the live raw log files and live model prediction files generated by this repository.

## Files Read By Logstash

Raw live logs:

```text
Firewall\live_logs\firewall_live.csv
WEB LOGS MODEL\live_logs\web_access.log
SSH\live_logs\ssh_auth.log
```

Live model predictions:

```text
Firewall\live_scores\firewall_scores.jsonl
WEB LOGS MODEL\live_scores\web_scores.jsonl
SSH\live_scores\ssh_scores.jsonl
```

Docker Desktop mounts those local Windows folders into the Logstash container as read-only bind mounts. Logstash tails the mounted files, so newly appended local lines are detected live.

The Logstash file inputs use non-persistent read offsets for this demo. For a clean live run, empty the local files with `python .\live\generate_logs.py --reset ...` before generating the new batch, then keep the listener and Logstash running while the generator appends lines.

## Start ELK

Open PowerShell from the repository root:

```powershell
.\elk\start_elk.ps1
```

`start_elk.ps1` creates `elk\.env` if it is missing and backfills any new required settings from `elk\.env.example`. The example file includes stable Kibana encryption keys so dashboards, actions, alerting, sessions, and reporting can use encrypted saved objects.

Install Elasticsearch index templates, Kibana data views, visualizations, and the dashboard:

```powershell
.\elk\setup_kibana.ps1
```

`setup_kibana.ps1` installs templates before Logstash starts. This avoids Logstash creating indices with weak automatic mappings.

Open Kibana:

```text
http://localhost:5601
```

Dashboard:

```text
SSH/Web/Firewall Live Security Dashboard
```

## Produce Live Data

PowerShell terminal 1:

```powershell
.\.venv\Scripts\Activate.ps1
python .\live\listen_and_score.py
```

PowerShell terminal 2:

```powershell
.\.venv\Scripts\Activate.ps1
python .\live\generate_logs.py --reset --count 100 --interval 0.5 --attack-rate 0.3
```

## Dashboard Panels

The setup script creates these Kibana panels:

```text
Prediction Trend by Model
Prediction Mix by Model
Average Risk by Model
Raw Log Volume by Type
Firewall Actions
Web Status Outcomes
SSH Login Status
Top SSH Source IP Risk
Firewall Destination Ports by Action
Web Error URLs by Status
SSH Usernames by Status
Risk Bands by Model
Sigma Rule Hits by Rule
Sigma Rule Hits by Severity
```

`Web Status Outcomes` uses the normalized `status_label` field:

```text
success, redirect, client_error, server_error, other
```

`SSH Login Status` uses the normalized `status_label` field:

```text
accepted, failed, invalid_user, auth_failure, connection, command, other
```

`Sigma Rule Hits` panels use prediction rows enriched by the Python rule engine in `live\sigma_rule_engine.py`.

## Reset Demo State

Stop containers:

```powershell
.\elk\stop_elk.ps1
```

Stop containers and remove Elasticsearch/Logstash volumes:

```powershell
.\elk\stop_elk.ps1 -Reset
```

Then start ELK again and rerun:

```powershell
.\elk\start_elk.ps1
.\elk\setup_kibana.ps1
```

## Manual Docker Commands

If you prefer not to use the helper scripts:

```powershell
docker compose --env-file .\elk\.env -f .\elk\docker-compose.yml up -d elasticsearch kibana
python .\elk\setup_elk.py
docker compose --env-file .\elk\.env -f .\elk\docker-compose.yml up -d logstash
docker compose --env-file .\elk\.env -f .\elk\docker-compose.yml down
```

## Troubleshooting

If PowerShell blocks local scripts for the current terminal session:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

If Kibana loads before the dashboard is installed, wait until `http://localhost:5601` is reachable and rerun:

```powershell
.\elk\setup_kibana.ps1
```

If charts look stale, hard refresh the browser and set the Kibana time picker to `Last 15 minutes` or a wider window.

If Kibana reports this error:

```text
Unable to create actions client because the Encrypted Saved Objects plugin is missing encryption key.
```

Pull the latest repo changes, then recreate Kibana with the updated `elk\.env` values:

```powershell
.\elk\stop_elk.ps1
.\elk\start_elk.ps1
.\elk\setup_kibana.ps1
```

The PowerShell scripts automatically append any missing keys from `elk\.env.example` into your existing `elk\.env`. If Kibana still uses old container state, reset the demo volumes:

```powershell
.\elk\stop_elk.ps1 -Reset
.\elk\start_elk.ps1
.\elk\setup_kibana.ps1
```

If Docker reports container name conflicts:

```powershell
.\elk\stop_elk.ps1
docker ps -a
```

Remove only this project's stopped containers if necessary:

```powershell
docker rm ssh-web-firewall-elasticsearch ssh-web-firewall-kibana ssh-web-firewall-logstash
```
