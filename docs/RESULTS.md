### Web

Web ROC-AUC is high, but PR-AUC is low. This is expected because pseudo anomalies are very rare:

```text
942 pseudo anomaly windows out of 182,278 total windows
```

The Web validation label is:

```text
error_rate > 0.5
```

This is useful for sanity checking but is not a full security ground truth.

## Figure Types

Every model has:

| Figure | Purpose |
| --- | --- |
| `confusion_matrix.png` | Shows normal/anomaly classification counts |
| `roc_curve.png` | Measures ranking quality across thresholds |
| `precision_recall_curve.png` | More informative under class imbalance |
| `score_distribution.png` | Shows anomaly score separation |
| `label_balance.png` | Shows normal/anomaly class balance |
| `feature_score_correlation.png` | Simple explanation of features associated with higher anomaly score |
| `inference_latency.png` | Measures scoring latency by batch size |

## Limitations

- Firewall labels are the strongest because they are derived from the dataset's `Action` field.
- Web labels are pseudo labels based on HTTP error-heavy windows.
- SSH labels are pseudo labels based on behavior rules.
- Isolation Forest produces anomaly rankings, not attack family classifications.
- Feature/score correlation is explanatory, but it is not causal proof.
